# Entrada manual (sem BM e/ou sem CRM)

Se o usuário não tem acesso à API do Meta e/ou ao CRM, **pergunte os números no
chat** e monte o relatório igual. Dá para misturar (Meta via API + comercial
manual, por exemplo). Pergunte só o que ele tiver — campo sem dado vira "—".

## Marketing (topo) — pergunte
- Período (ex.: 28/05–24/06) e **investido** no período (R$)
- **Impressões**, **cliques** (ou CTR), **leads** gerados
- Se tiver: CPL, gasto por plataforma (Instagram x Facebook), por região,
  desempenho por criativo/anúncio, frequência

## Comercial (fundo) — pergunte
- **MQL** (leads qualificados), **agendamentos**, **R1 realizadas** (e no-show)
- **Qualificadas / R2**, **propostas**, **contratos** (vendas)
- **Pipeline ativo**: para cada deal em aberto — nome, estágio (quente/morno/frio)
  e uma observação curta do que está travando/avançando
- Se tiver: **valor** dos contratos/deals (essencial para ROI/ROAS — ver abaixo)

## Sempre vale perguntar (melhora muito o relatório)
- **Ticket / valor médio do contrato** e **modelo** (fee mensal, % etc.) → permite
  calcular **ROI/ROAS/CAC**, não só volume.
- **Metas** do período (ex.: meta de MQL, de vendas) → para comparar realizado x meta.
- **Período anterior** (mesmos números) → para mostrar **evolução**.

## Branding (opcional)
- **Logo** (arquivo/URL) e **cores da marca** (primária e secundária). Sem isso, o
  relatório sai sem logo e com as cores genéricas (navy + ouro/teal). Ver a skill
  `gerar-relatorio-funil` → `references/branding.md`.
